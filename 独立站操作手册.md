## 1. 项目概述

**MAISON HAN**（漢館）是一个高端烈酒电商独立站，主营稀有茅台、日本威士忌、干邑、葡萄酒及香槟。

| 属性 | 说明 |
|------|------|
| 网站域名 | https://www.boozett.top |
| 前端技术 | 纯静态 HTML + 原生 JavaScript（无构建框架） |
| 后端服务 | Cloudflare Pages Functions（Serverless） |
| 支付网关 | Stripe（Checkout Session 模式） |
| 邮件服务 | Resend API |
| 部署平台 | Cloudflare Pages（主） / Netlify（备） |

## 2. 技术架构

```
┌──────────────────────────────────────────────┐
│               用户浏览器                      │
│   index.html（单页，5000+ 行）                │
│   · 购物车（localStorage）                   │
│   · 多语言 i18n（4 种语言）                  │
│   · 多货币显示（7 种货币）                   │
│   · 年龄验证 Age Gate                        │
└────────────────┬─────────────────────────────┘
                 │ HTTP
┌────────────────▼─────────────────────────────┐
│         Cloudflare Pages Functions            │
│                                               │
│  GET  /list-products          商品列表        │
│  POST /create-checkout-session  创建支付会话  │
│  POST /stripe-webhook           支付回调      │
│  POST /contact                  联系表单      │
│  POST /newsletter-subscribe     邮件订阅      │
│  中间件 _middleware.js          OG 标签注入   │
└──────────────┬──────────────┬────────────────┘
               │              │
    ┌──────────▼──┐    ┌──────▼──────┐
    │  Stripe API  │    │  TaTa 后端   │
    │  支付 + 商品  │    │  商品目录    │
    └─────────────┘    └─────────────┘
```

### 商品数据来源优先级（三级 fallback）

```
优先级 1：MAISON_HAN_API_BASE（TaTa 后端，生产环境推荐）
     ↓ 失败或未配置
优先级 2：STRIPE_SECRET_KEY（直接读 Stripe 商品库）
     ↓ 失败或未配置
优先级 3：/catalog.json（站内静态文件，演示/离线模式）
```

## 4. 本地开发

### 第一步：创建本地环境变量文件

在项目根目录创建 `.dev.vars` 文件（**切勿提交到 Git**）：

```ini
# .dev.vars（Cloudflare Pages 本地开发专用）

# 必填：Stripe 测试密钥
STRIPE_SECRET_KEY=sk_test_xxxxxxxxxxxxxxxxxxxxxxxx

# 可选：TaTa 后端地址（留空则走 Stripe 直拉）
MAISON_HAN_API_BASE=https://api.boozett.top/storefront/products

# 可选：强制使用 catalog.json 静态数据（1/true/yes）
STATIC_CATALOG=

# 可选：Stripe Webhook 签名密钥（本地测试 webhook 用）
STRIPE_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxxxxx

# 可选：Resend 邮件 API Key
RESEND_API_KEY=re_xxxxxxxxxxxxxxxx
```

### 第二步：启动本地服务

```bash
npm run pages:dev
```

服务启动后访问 `http://localhost:8788`

> **说明**：`wrangler pages dev .` 会同时运行静态文件服务和 Functions，行为与线上 Cloudflare Pages 完全一致。

### 本地调试要点

- 修改 `index.html` 后**刷新浏览器**即可生效（无需重启服务）
- 修改 `functions/` 下的函数文件后，wrangler 会**自动热重载**
- 购物车数据存储在浏览器 `localStorage`（键名：`han_cart`）
- 语言偏好存储键名：`han_lang`；货币偏好：`han_curr`

## 5. 环境变量配置

### Cloudflare Pages 配置方式

登录 [Cloudflare Dashboard](https://dash.cloudflare.com) → Pages → 你的项目 → Settings → Environment Variables

### 变量说明

| 变量名 | 是否必填 | 说明 |
|--------|----------|------|
| `STRIPE_SECRET_KEY` | **必填** | Stripe 密钥。测试：`sk_test_...`；生产：`sk_live_...` |
| `STRIPE_WEBHOOK_SECRET` | 强烈建议 | Stripe Webhook 签名验证密钥，防止伪造回调。`whsec_...` |
| `RESEND_API_KEY` | 发邮件必填 | Resend 邮件服务密钥，用于订单确认邮件和联系表单通知 |
| `MAISON_HAN_API_BASE` | 可选 | TaTa 后端根地址，**不带末尾斜杠**。例：`https://api.boozett.top` |
| `STATIC_CATALOG` | 可选 | 设为 `1` 时强制使用 `/catalog.json`，忽略 TaTa 和 Stripe |
| `PRODUCT_IMAGE_BASE` | 可选 | 商品图片相对路径的前缀域名（仅 Stripe 直拉模式需要） |

### Stripe 密钥切换（测试 → 生产）

1. 在 Stripe Dashboard 获取 `sk_live_...` 密钥
2. 在 Cloudflare Pages **生产环境**的 Environment Variables 中更新 `STRIPE_SECRET_KEY`
3. 重新部署一次即可生效

> ⚠️ **安全提示**：绝对不能把 `STRIPE_SECRET_KEY` 硬编码到任何前端文件或提交到 Git。

## 6. 商品管理

### 方式一：通过 Stripe Dashboard 管理（推荐生产使用）

每个商品是 Stripe 上的一个 Product，需在 Metadata 中设置以下字段：

| Metadata 字段 | 必填 | 说明 | 示例值 |
|---------------|------|------|--------|
| `maison_han` | **必填** | 标记为本站商品，必须为 `true` | `true` |
| `maison_han_id` | **必填** | 正整数，商品唯一 ID | `101` |
| `cat` | **必填** | 商品分类 | `baijiu` / `whisky` / `wine` / `sparkling` / `spirits` / `cognac` / `others` |
| `sort_order` | 建议填 | 排列顺序，数字越小越靠前 | `10` |
| `bv` | 可选 | 卡片背景色主题 | `bv-maotai` / `bv-japan` / `bv-scotch` / `bv-wine` / `bv-cognac` |
| `bottle` | 可选 | 瓶型 SVG 标识 | `maotai-vintage` / `hibiki-21` |
| `abv` | 可选 | 酒精度 | `53%` |
| `vol` | 可选 | 容量 | `500 ml` |
| `origin` | 可选 | 产地 | `Guizhou, China` |
| `year` | 可选 | 年份 | `1988` |
| `badge` | 可选 | 角标文字 | `Vintage Reserve` |
| `sub` | 可选 | 副标题（80字符以内） | `Single-bottle reserve.` |
| `name_html` | 可选 | 支持 HTML 的商品名（含 `<em>` 斜体） | `<em>Moutai</em> Feitian` |
| `display_no` | 可选 | 编号展示 | `047` |
| `cat_label` | 可选 | 分类标签展示文字 | `Chinese baijiu` |

### 方式二：编辑 catalog.json（静态演示模式）

适用于：无 Stripe / 离线演示 / 快速预览

文件路径：`/catalog.json`

字段结构示例：

```json
{
  "id": 101,
  "productId": "",
  "priceId": null,
  "no": "047",
  "cat": "spirits",
  "catLabel": "Chinese baijiu",
  "bv": "bv-maotai",
  "bottle": "maotai-vintage",
  "image": "images/maotai-vintage.png",
  "name": "<em>Moutai</em> Feitian &middot; 1988",
  "sub": "Single-bottle reserve.",
  "price": 24800,
  "abv": "53%",
  "vol": "500 ml",
  "origin": "Guizhou, China",
  "year": "1988",
  "badge": "Vintage Reserve",
  "desc": "商品详情描述文字",
  "sortOrder": 10
}
```

> **注意**：`priceId` 为 `null` 时，商品能正常展示但**无法结账**。如需结账须填入 Stripe Price ID。

### 商品图片管理

- 图片存放于 `/images/` 目录
- 支持格式：`.jpg` `.png` `.webp`
- 图片命名建议使用英文小写 + 连字符
- 替换图片后需**重命名**（因 Cloudflare 会缓存 1 年）：`Cache-Control: public, max-age=31536000, immutable`

## 8. API 接口说明

### GET /list-products

获取商品列表，服务端缓存 60 秒。

**请求**：无参数

**响应示例**：

```json
[
  {
    "id": 101,
    "productId": "prod_xxx",
    "priceId": "price_xxx",
    "no": "047",
    "cat": "spirits",
    "catLabel": "Chinese baijiu",
    "bv": "bv-maotai",
    "bottle": "maotai-vintage",
    "image": "https://...",
    "name": "<em>Moutai</em> Feitian · 1988",
    "sub": "Single-bottle reserve.",
    "price": 24800,
    "abv": "53%",
    "vol": "500 ml",
    "origin": "Guizhou, China",
    "year": "1988",
    "badge": "Vintage Reserve",
    "desc": "...",
    "sortOrder": 10
  }

```

**响应头**：
- `X-Cache: HIT/MISS` — 是否命中缓存
- `X-Catalog-Source: tata/stripe/static` — 数据来源

---

### POST /create-checkout-session

创建 Stripe Checkout 支付会话。

**请求 Body**：

```json
{
  "items": [
    {
      "name": "Moutai Feitian 1988",
      "sub": "Single-bottle reserve.",
      "priceUsd": 24800,
      "qty": 1,
      "image": "https://..."
    }
  ],
  "email": "buyer@example.com",
  "ageConfirmed": true,
  "lang": "en"
}
```

**响应**：

```json
{ "url": "https://checkout.stripe.com/c/pay/cs_live_xxx" }
```

前端收到 `url` 后直接跳转即可。

**价格限制**：单件商品最高 $50,000 USD；每单最多 50 件商品。

**支持收货国家（26个）**：
`US CA GB AU NZ JP HK SG TW MO DE FR IT ES NL BE SE DK FI NO CH AT IE PT LU PL`

**运费**：固定 **$50 USD**（保险快递，3-10 个工作日）

---

### POST /stripe-webhook

Stripe 支付完成后的异步回调，需在 Stripe Dashboard 配置 Webhook 端点。

**Webhook 端点 URL**：`https://www.boozett.top/stripe-webhook`

**监听事件**：`checkout.session.completed`

**触发流程**：
1. 验证 Stripe 签名（防止伪造）
2. 查找或创建 Stripe Customer 档案
3. 同步订单数据到 TaTa 后端（如配置了 `MAISON_HAN_API_BASE`）
4. 通过 Resend 发送多语言确认邮件给买家

**Stripe Dashboard 配置步骤**：

1. 进入 Stripe Dashboard → Developers → Webhooks → Add endpoint
2. 填入端点：`https://www.boozett.top/stripe-webhook`
3. 选择事件：`checkout.session.completed`
4. 复制 Signing secret（`whsec_...`）填入环境变量 `STRIPE_WEBHOOK_SECRET`

---

## 9. 前端功能说明

### 年龄验证（Age Gate）

- 首次访问强制弹出，要求输入出生年份（4 位数字）
- 最低年龄：法定饮酒年龄（各地不同，前端按 21 岁校验）
- 勾选"记住此设备"后，验证结果存入 `localStorage`（键名：`han_age_ok`）
- 下次访问若存在有效记录，自动跳过验证

### 购物车

- 数据存储：浏览器 `localStorage`（键名：`han_cart`）
- 支持增减数量、删除单品
- 右侧抽屉式展示
- 结账前需勾选年龄确认复选框

### 商品详情（PDP）

- 点击商品卡片打开全屏详情页
- URL 会更新为 `/#/product/{id}` 格式（支持直接访问）
- 分享按钮生成 `/?pid={id}` 格式链接（适合社交分享）

### 搜索功能

- 点击顶部 Search 按钮触发
- 实时过滤商品名称、副标题、产地等字段

### 分类筛选

- 商品列表上方有分类导航标签（All / Spirits / Baijiu / Whisky / Wine / Sparkling）
- 点击可实时过滤显示对应分类
- 可通过 URL hash 或链接 `data-cat-link` 属性跳转并自动激活分类

## 10. 多语言系统

### 支持语言

| 语言代码 | 显示名称 | 触发条件 |
|----------|----------|----------|
| `en` | English | 默认 |
| `zh-CN` | 中文（简体） | 浏览器语言 zh-CN |
| `zh-TW` | 繁體中文 | 浏览器语言 zh-TW |
| `ja` | 日本語 | 浏览器语言 ja |

### 切换方式

1. 页面右上角语言下拉面板手动切换
2. 浏览器语言自动检测（首次访问）
3. 存储在 `localStorage`（键名：`han_lang`）

### 新增/修改翻译

翻译文本分布在两个位置：

**1. `index.html` 内的 `I18N` 对象**（主页面所有文字）

搜索 `const I18N = {` 定位，按 `en` 的 key 增减对应语言的翻译。

**2. `i18n.js` 的 `DICT` 对象**（success.html、cancel.html、account.html 共用）

### 货币显示

支持 7 种货币（仅前端汇率换算展示，Stripe 实际收款均为 USD）：

`USD · EUR · GBP · JPY · HKD · SGD · CNY`

汇率数据内嵌于 `index.html` 的 `CURRENCY` 对象，需定期手动更新。

## 11. 支付与结账流程

```
用户点击 Checkout
      │
      ▼
前端收集购物车（商品名、价格、数量）
      │
      ▼
POST /create-checkout-session
（传入 items、email、ageConfirmed、lang）
      │
      ▼
服务端验证参数 → 查找/创建 Stripe Customer
      │
      ▼
调用 Stripe API 创建 Checkout Session
（price_data 动态定价，固定 $50 运费）
      │
      ▼
返回 Stripe Checkout URL
      │
      ▼
前端跳转至 Stripe 托管支付页面
      │
      ├─ 支付成功 → 跳转 /success.html?session_id=xxx
      └─ 取消支付 → 跳转 /cancel.html
            │
            ▼（异步，Stripe 回调）
      POST /stripe-webhook
      · 建档 Stripe Customer
      · 同步 TaTa 订单
      · 发送确认邮件
```

### 支付安全设计

- 前端**不传 priceId**，直接传商品名和价格（USD 金额），杜绝 priceId 枚举攻击
- 单价上限 $50,000；数量上限 99
- 结账时要求买家填写收货地址、电话
- Webhook 使用 HMAC-SHA256 签名验证，5 分钟内有效

## 12. 邮件系统

所有邮件均通过 **Resend** 服务发送，发件地址：`orders@boozett.top`

### 邮件类型

| 场景 | 触发时机 | 收件人 |
|------|----------|--------|
| 订单确认 | Stripe Webhook 收到 `checkout.session.completed` | 买家 |
| 联系表单通知 | 用户提交 /contact 表单 | 运营邮箱 |
| 联系表单自动回复 | 用户提交 /contact 表单 | 用户本人 |

### 订单确认邮件

支持 4 种语言（由订单 metadata 中的 `lang` 字段决定）：
- 英文（`en`）
- 简体中文（`zh-CN`）
- 繁体中文（`zh-TW`）
- 日文（`ja`）

邮件内容包含：订单编号、扣款金额、收件人姓名、返回网站按钮。

### 配置 Resend

1. 注册 [Resend](https://resend.com) 账号
2. 验证发件域名 `boozett.top`（需添加 DNS TXT/MX 记录）
3. 获取 API Key，填入环境变量 `RESEND_API_KEY`

## 13. 分享链接功能

### 工作原理

| 场景 | URL 格式 | 说明 |
|------|----------|------|
| 用户自己访问商品详情 | `/#/product/101` | Hash 路由，不被爬虫识别 |
| 点击分享按钮 | `/?pid=101` | Query 参数，爬虫可读 |

### 分享链接流程

1. 用户打开商品详情，点击"分享"按钮
2. 生成 `https://www.boozett.top/?pid=101` 格式链接
3. 若浏览器支持 `navigator.share`（手机端），调用系统分享面板（含标题、副标题、价格）
4. 否则自动复制链接到剪贴板，提示"Link copied"

### 访问分享链接

1. 访客打开 `/?pid=101`
2. 普通访客：JS 检测 `?pid` 参数 → 等待商品数据加载完成 → 自动打开商品详情弹窗 → 清除 URL 参数
3. 爬虫（微信/WhatsApp 等）：`_middleware.js` 拦截 → 从 TaTa 拉取商品信息 → 注入动态 OG 标签 → 返回带正确 title/image 的 HTML

## 14. 常见问题排查

### Q1：商品列表加载为空

**检查顺序**：

1. 打开浏览器控制台，查看 `GET /list-products` 的响应状态
2. 若返回 500：检查 `STRIPE_SECRET_KEY` 是否正确配置
3. 若返回 502：检查 `MAISON_HAN_API_BASE` 的 TaTa 后端是否可访问
4. 若响应为空数组 `[]`：检查 Stripe 商品是否设置了 `maison_han=true` 的 metadata
5. 临时解决：设置 `STATIC_CATALOG=1`，商品将从 `catalog.json` 读取

---

### Q2：结账后支付页面报错

**可能原因**：

- `STRIPE_SECRET_KEY` 是测试密钥但商品是生产 Price ID（或反之）
- 单价超过 $50,000 限制
- 购物车为空

**调试方法**：在 Cloudflare Pages 的 **Functions** → **Logs** 中查看 `create-checkout-session` 的错误日志。

---

### Q3：Webhook 不触发 / 邮件发送失败

**检查步骤**：

1. 确认 Stripe Dashboard → Webhooks 中端点 URL 正确（`https://www.boozett.top/stripe-webhook`）
2. 确认 `STRIPE_WEBHOOK_SECRET` 与 Dashboard 上的 Signing secret 一致
3. 在 Stripe Dashboard → Webhooks → 你的端点，查看最近事件的投递状态
4. 若 webhook 成功但邮件未发送：检查 `RESEND_API_KEY` 是否配置，Resend 控制台查看发送记录

---

### Q4：分享链接在微信中显示为纯文本链接（无卡片）

**原因**：微信强缓存；或 OG 注入未生效。

**解决方法**：

1. 确认 `MAISON_HAN_API_BASE` 已正确配置（中间件需要拉取商品数据）
2. 在链接后加 `?v=2`（如 `/?pid=101&v=2`）强制刷新微信缓存
3. 等待 24 小时后重试（微信卡片缓存最长 24 小时）
4. 使用微信官方"清除缓存"工具：`http://debugx5.qq.com`

---

### Q5：年龄验证跳过不了 / 反复弹出

**原因**：浏览器隐私模式（无痕模式）不保存 localStorage。

**解决**：使用普通浏览器窗口，勾选"记住此设备"。

**开发调试跳过年龄验证**：
```javascript
// 在浏览器控制台执行
localStorage.setItem('han_age_ok', '1');
location.reload();
```

---

### Q6：语言/货币切换后刷新页面恢复默认

**检查** `localStorage` 是否正常读写：

```javascript
// 控制台检查
console.log(localStorage.getItem('han_lang'));    // 应显示 'zh-CN' 等
console.log(localStorage.getItem('han_curr'));    // 应显示 'CNY' 等
```

若为 `null`，说明浏览器禁用了 localStorage（隐私设置或安全策略）。

---

### Q7：本地开发时函数提示 "Missing STRIPE_SECRET_KEY"

确认以下几点：

1. `.dev.vars` 文件在**项目根目录**（与 `package.json` 同级）
2. 文件格式正确（每行 `KEY=VALUE`，无引号，无空格）
3. 重新执行 `npm run pages:dev`（修改 `.dev.vars` 需重启）
